# 开发环境问题排查记录

## 问题描述

API 服务启动后无法正常响应请求，应用卡在启动阶段。

## 现象

1. 执行 `./dev.sh start api` 后，进程显示已启动
2. `curl https://localhost:44305/api/abp/application-configuration` 无响应
3. 端口 44305 未被监听
4. 日志文件只显示 `[INF] Starting KnowledgeHub.HttpApi.Host.` 后无更多输出
5. 进程 CPU 占用高，似乎卡在某个循环中

## 排查过程

### 1. 检查数据库连接
```bash
PGPASSWORD=postgres psql -h localhost -p 5433 -U postgres -d KnowledgeHub -c "SELECT 1"
# 结果：数据库连接正常
```

### 2. 检查 HTTPS 证书
```bash
dotnet dev-certs https --check
# 结果：No valid certificate found
```

尝试创建证书失败：
```bash
dotnet dev-certs https --trust
# 结果：There was an error creating the HTTPS developer certificate
```

错误信息：`CSSM_ModuleLoad(): One or more parameters passed to a function were not valid`

### 3. 尝试切换到 HTTP
- 修改 `launchSettings.json` 使用 `http://localhost:5001`
- 修改 `appsettings.json` 中的 `App:SelfUrl` 和 `AuthServer:Authority`
- 修改 Angular `environment.ts` 中的 API URL

结果：问题依旧，API 仍然无法正常启动

### 4. 检查进程状态
```bash
sample <pid> 3
```
进程似乎卡在无限循环中

## 可能的原因

1. **macOS 安全框架问题**：`CSSM_ModuleLoad()` 错误可能是 macOS 安全相关的系统问题
2. **.NET 10 兼容性问题**：项目使用 .NET 10 (预览版)，可能存在兼容性问题
3. **依赖注入或模块初始化问题**：ABP 模块初始化过程中可能卡住

## 待尝试的解决方案

1. 重新安装 .NET SDK
2. 检查 macOS 钥匙串状态，尝试重置
3. 在其他环境（Linux/Docker）中测试
4. 回退到 .NET 8 或 .NET 9 LTS 版本
5. 添加更多诊断日志，定位卡住的具体位置

- .NET SDK 10.0.102
