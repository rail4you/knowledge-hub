# 角色管理租户功能修复记录

## 问题描述

角色管理页面需要支持租户关联功能，包括：
1. 按租户过滤角色列表
2. 创建角色时选择所属租户
3. 显示角色所属租户
4. 中文本地化

## 解决过程

### 问题1：侧边栏点击角色菜单，显示的是ABP默认组件而非自定义组件

**原因分析：**
- `app.routes.ts` 中 `identity` 的 `loadChildren` 会匹配所有 `/identity/**` 路径
- 第82-88行定义的 `/identity/roles` 自定义路由被 ABP 默认模块覆盖

**解决方案：**
使用 ABP 的 `ReplaceableComponentsService` 替换内置的 `RolesComponent`

**实现步骤：**
1. 创建 `angular/src/app/identity-roles.config.ts`：
```typescript
import { provideAppInitializer, inject } from '@angular/core';
import { ReplaceableComponentsService } from '@abp/ng.core';
import { eIdentityComponents } from '@abp/ng.identity';
import { IdentityRolesComponent } from './admin/identity-roles/identity-roles.component';

function initIdentityRoles() {
  const replaceableComponents = inject(ReplaceableComponentsService);
  replaceableComponents.add({
    key: eIdentityComponents.Roles,
    component: IdentityRolesComponent,
  });
}

export const IDENTITY_ROLES_PROVIDER = [
  provideAppInitializer(() => {
    initIdentityRoles();
  }),
];
```

2. 在 `app.config.ts` 中引入该 provider
3. 删除重复的路由定义

### 问题2：新建/编辑对话框显示空白

**原因分析：**
- 缺少 `NzCheckboxModule` 导入
- modal 内容的 `@if` 条件问题

**解决方案：**
1. 在组件中导入 `NzCheckboxModule`
2. 修改 modal 模板，使用 `*ngIf="form"` 确保 form 存在后再渲染

### 问题3：中文字符串未翻译

**原因分析：**
- 部分 localization key 只存在于 `AbpIdentity` 资源中
- ABP 默认优先查找项目自定义资源（如 `KnowledgeHub`）
- `getRoleDisplayName` 方法未使用 `::` 前缀

**解决方案：**
1. 在 `KnowledgeHub` 资源中添加缺失的 localization keys：
```json
// zh-Hans.json
{
  "Tenant": "租户",
  "FilterByTenant": "按租户过滤",
  "All": "全部",
  "SelectTenant": "选择租户",
  "Global": "全局",
  "Roles": "角色",
  "NewRole": "新角色",
  "DisplayName:RoleName": "角色名称",
  "DisplayName:IsDefault": "默认",
  "DisplayName:IsPublic": "公开",
  "RoleName:admin": "管理员"
}
```

2. 修改 `getRoleDisplayName` 方法：
```typescript
getRoleDisplayName(roleName: string | undefined): string {
  if (!roleName) return '';
  const translated = this.localization.instant(`::RoleName:${roleName}`);
  return translated || roleName;
}
```

## 最终效果

1. ✅ 角色列表显示中文角色名称（企业用户、联盟管理员、院校管理员、学生、教师、管理员）
2. ✅ 租户过滤下拉框正常工作（全部、全局、学校）
3. ✅ 新建角色对话框显示租户选择下拉框
4. ✅ 编辑角色对话框正常显示角色信息
5. ✅ 所有界面文本正确显示中文

## 涉及的文件

### 修改文件

## 关键技术点

1. **ABP 组件替换机制**：使用 `ReplaceableComponentsService` 可以替换 ABP 内置组件
2. **Localization 查找顺序**：ABP 会先查找项目自定义资源，再查找框架资源
3. **Localization key 前缀**：使用 `::` 前缀可以让 ABP 从所有资源中查找 key
4. **Angular standalone component**：需要手动导入所有依赖的 NgModule
