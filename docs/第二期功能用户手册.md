# 第二期功能用户手册

## 1. 文档说明

本文档用于说明系统第二期已交付功能的使用方式，适用于：

- 联盟管理员
- 院校管理员
- 教师
- 学生
- 企业用户

本文档仅覆盖当前已实现并可交付的第二期功能，不包含已明确延后的实时直播、实时音视频面试、屏幕共享、会议录制等能力。

## 2. 第二期功能范围

第二期当前已实现模块如下：

| 模块 | 主要功能 |
| --- | --- |
| 双高专业群评估支撑 | 项目管理、指标配置、自动采集、手工填报、佐证材料、报告导出 |
| 微专业 | 微专业创建、课程绑定、学生报名、进度汇总、证书发放 |
| 就业 | 企业档案、岗位发布、简历管理、岗位投递、面试预约、指导记录、就业去向、统计导出 |
| 资讯 | 分类管理、资讯发布、审核发布、前台浏览、评论互动、后台管理 |
| 实训 | 实训项目、任务拆解、资料配置、学生参与、提交、指导、评分、导出 |

## 3. 登录与角色

### 3.1 角色说明

| 角色 | 典型职责 |
| --- | --- |
| 联盟管理员 | 跨院校查看与管理、审核与统计汇总 |
| 院校管理员 | 本校功能配置、内容审核、项目与统计管理 |
| 教师 | 微专业、实训、指导、评分、部分管理操作 |
| 学生 | 学习、报名、投递、查看结果与个人记录 |
| 企业用户 | 维护企业档案、发布岗位、预约面试 |

### 3.2 登录后菜单

第二期新增菜单主要包括：

- 评估
- 微专业
- 就业
- 资讯
- 实训

不同角色看到的菜单和按钮会因权限不同而有所差异。

## 4. 双高专业群评估支撑

### 4.1 功能目标

支持院校按批次建立双高评估项目，配置指标，采集业务数据，补录人工数据，维护佐证材料，并导出报表。

### 4.2 适用角色

- 联盟管理员
- 院校管理员
- 指定教师或项目负责人

### 4.3 功能入口

- 项目列表：`/assessment/double-high/projects`
- 项目详情：`/assessment/double-high/project/:id`
- 报告中心：`/assessment/double-high/report-center`
- 后台管理：`/admin/assessment/double-high`

### 4.4 主要操作

#### 4.4.1 新建评估项目

1. 进入“双高管理”页面。
2. 点击“新增项目”。
3. 填写项目标题、批次编号、起止时间、状态。
4. 配置指标清单、指标编码、分类、权重、目标值。
5. 保存项目。

#### 4.4.2 数据采集

系统当前支持从以下模块自动采集部分数量型指标：

- 资源数量
- 课程数量
- 微专业数量
- 微专业报名量
- 实训项目数量
- 实训报名量
- 资讯数量

操作方式：

1. 打开项目详情。
2. 点击“采集数据”。
3. 系统自动写入最新采集快照。

#### 4.4.3 手工填报

对于人工型指标，可在项目详情中手工录入数值和备注说明。

#### 4.4.4 佐证材料维护

支持三类佐证方式：

- 绑定系统资源
- 填写附件链接
- 填写外部链接

#### 4.4.5 报告导出

在项目详情或报告中心可导出 Excel 报表，用于汇报和留档。

## 5. 微专业

### 5.1 功能目标

在现有课程体系基础上，建设面向特定能力培养方向的微专业，形成课程集合、报名、学习进度、证书发放的一体化流程。

### 5.2 适用角色

- 联盟管理员
- 院校管理员
- 教师
- 学生

### 5.3 功能入口

- 微专业列表：`/micro-majors`
- 微专业详情：`/micro-majors/:id`
- 我的微专业：`/my/micro-majors`
- 微专业管理：`/admin/micro-majors`

### 5.4 主要操作

#### 5.4.1 创建微专业

1. 进入“微专业管理”。
2. 新建微专业，填写名称、简介、封面、行业方向、合作单位等信息。
3. 绑定课程列表。
4. 设置状态和证书规则后保存。

#### 5.4.2 学生报名

1. 学生进入微专业详情页。
2. 查看课程组成与要求。
3. 点击“报名”。
4. 系统生成报名记录。

#### 5.4.3 学习进度查看

学生在“我的微专业”中查看：

- 已报名微专业
- 完成课程数
- 完成率
- 当前状态

#### 5.4.4 证书发放

管理员或教师可对达到条件的学生发放证书，学生可在个人页查看证书信息。

## 6. 就业

### 6.1 功能目标

围绕“企业发布岗位 - 学生维护简历 - 投递 - 预约面试 - 指导 - 去向统计”形成标准就业服务链路。

### 6.2 适用角色

- 企业用户
- 学生
- 教师
- 院校管理员
- 联盟管理员

### 6.3 功能入口

- 岗位大厅：`/employment/jobs`
- 岗位详情：`/employment/jobs/:id`
- 企业档案：`/employment/employer-profile`
- 我的简历：`/employment/my-resumes`
- 我的投递：`/employment/my-applications`
- 就业指导：`/employment/my-guidance`
- 岗位管理：`/admin/employment/jobs`
- 面试管理：`/admin/employment/interviews`
- 就业统计：`/admin/employment/statistics`

### 6.4 主要操作

#### 6.4.1 企业档案维护

企业用户首次使用时，应先维护：

- 企业名称
- 统一社会信用代码
- 联系人
- 电话与邮箱
- 所属行业
- 合作院校
- 备注说明

#### 6.4.2 岗位发布

1. 企业用户进入岗位管理。
2. 新建岗位，填写岗位名称、描述、地区、薪资、技能要求、联系方式等。
3. 保存后进入草稿或待审状态。
4. 管理员审核通过后可发布。

#### 6.4.3 简历管理

学生可维护多版本简历，包括：

- 基本信息
- 专业与年级
- 个人简介
- 技能标签
- 项目经历
- 简历附件链接

可设置默认简历，用于投递时快速选择。

#### 6.4.4 岗位投递

1. 学生进入岗位详情。
2. 选择简历版本。
3. 填写补充说明。
4. 点击投递。

#### 6.4.5 面试预约

企业或管理方可在面试管理页：

- 选择投递记录
- 录入面试官
- 填写时间
- 填写地点或外部会议链接
- 填写备注

#### 6.4.6 面试结果回填

面试完成后可回填：

- 通过
- 待补充
- 未通过
- 面试纪要
- 结果说明

#### 6.4.7 就业指导

学生可查看教师或管理员录入的指导记录，也可通过系统现有 AI 职业规划能力获得辅助建议。

#### 6.4.8 就业去向与统计

可登记学生就业或升学去向，并按：

- 专业
- 年级
- 去向状态

进行统计和 Excel 导出。

## 7. 资讯

### 7.1 功能目标

实现资讯内容的分类、发布、审核、浏览和互动，支撑校内通知、行业动态、企业资讯等内容发布。

### 7.2 适用角色

- 管理员
- 教师
- 学生
- 企业用户

### 7.3 功能入口

- 资讯列表：`/news`
- 资讯详情：`/news/:id`
- 资讯管理：`/admin/news`

### 7.4 主要操作

#### 7.4.1 分类管理

管理员可配置资讯分类层级。

#### 7.4.2 资讯发布

可维护：

- 标题
- 摘要
- 正文
- 封面
- 标签
- 置顶/热门标记
- 评论开关

#### 7.4.3 审核与发布

资讯进入草稿或待审核状态后，由有权限人员审核发布。

#### 7.4.4 前台浏览与互动

用户可浏览已发布资讯，查看热门内容，进入详情页阅读，并进行评论或互动。

## 8. 实训

### 8.1 功能目标

通过项目化实训方式，支持任务拆解、资料配置、学生参与、提交、教师指导、评分和导出。

### 8.2 适用角色

- 管理员
- 教师
- 学生

### 8.3 功能入口

- 实训项目列表：`/practicum/projects`
- 实训项目详情：`/practicum/project/:id`
- 我的实训：`/practicum/my`
- 实训管理：`/admin/practicum/projects`

### 8.4 主要操作

#### 8.4.1 新建实训项目

可维护：

- 项目名称
- 项目简介
- 绑定课程
- 面向专业/班级
- 项目时间
- 总分
- 是否允许反复提交

#### 8.4.2 任务拆解

每个实训项目可配置多个任务，包括：

- 任务标题
- 任务说明
- 要求
- 截止时间
- 分值权重

#### 8.4.3 资料配置

支持配置指南、案例、模板、外部链接等资料。

#### 8.4.4 学生参与与提交

学生进入项目后可报名参与，并针对各任务进行多次提交。

#### 8.4.5 教师指导与评分

教师可对学生提交内容进行：

- 指导记录
- 反馈
- 评分
- 评价说明

#### 8.4.6 导出

支持导出实训成绩与过程记录。

## 9. 典型使用流程

### 9.1 教师侧流程

1. 创建微专业或实训项目。
2. 配置课程、任务或资料。
3. 查看学生报名与学习/实训进度。
4. 录入指导、评分或证书。

### 9.2 学生侧流程

1. 报名微专业或参与实训。
2. 完成学习与任务提交。
3. 维护简历并投递岗位。
4. 查看面试安排、指导记录和结果。

### 9.3 企业侧流程

1. 完善企业档案。
2. 发布岗位。
3. 查看投递。
4. 预约面试。
5. 回填面试结果。

## 10. 已知边界

当前第二期未纳入：

- 实时视频直播
- 实时音视频面试
- 屏幕共享
- 面试录制
- 高并发 RTC 调度

如后续进入第三阶段或专项扩展，可在当前架构基础上继续建设。

## 11. 建议的上线后准备

- 为各角色创建测试账号和正式账号
- 为双高、微专业、实训、就业分别准备基础演示数据
- 按角色进行一次菜单与权限检查
- 在正式培训前完成数据库迁移与环境验证

## 12. 结语

第二期当前已经完成双高评估、微专业、就业、资讯、实训五个核心模块的首版交付。建议在正式使用前，结合验收文档进行逐项演示和验证。
