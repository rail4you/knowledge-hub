## 问题 1: 创建用户时无法保存租户信息

### 问题描述
在 `/identity/users` 页面创建新用户时，虽然界面上可以选择租户，但保存后租户信息丢失。

### 原因分析
`identity-form-prop-contributors.ts` 中 `tenantId` 字段设置了 `isExtra: false`，导致租户 ID 不会被放入 `extraProperties` 中。而 `CustomIdentityUserService` 期望从 `extraProperties` 中读取租户 ID。

### 解决方案
```typescript
// angular/src/app/identity-form-prop-contributors.ts
const tenantProp = new FormProp<IdentityUserDto>({
  type: ePropType.String,
  name: 'tenantId',
  displayName: '所属租户',
  isExtra: true,  // 改为 true
  // ...
});
```

- Commit: `a9942fc`
- Message: `fix: enable tenant selection when creating new users`

## 问题 2: DbMigrator 镜像无法运行

### 问题描述
DbMigrator 容器启动时报错：
```
Could not execute because the specified command or file was not found.
```

### 原因分析
`Dockerfile.local` 使用 `sdk` 镜像作为运行时，但应该使用 `aspnet` 运行时镜像。

### 解决方案
```dockerfile
# 修改前
FROM mcr.microsoft.com/dotnet/sdk:10.0
USER $APP_UID
COPY --from=build /app/publish .
WORKDIR /app
ENTRYPOINT ["dotnet", "KnowledgeHub.DbMigrator.dll"]

# 修改后
FROM mcr.microsoft.com/dotnet/aspnet:10.0
WORKDIR /app
COPY --from=build /app/publish .
ENTRYPOINT ["dotnet", "KnowledgeHub.DbMigrator.dll"]
```

### 文件位置
- `src/KnowledgeHub.DbMigrator/Dockerfile.local`

## 问题 3: HTTPS 证书问题导致 API 无法访问

### 问题描述
前端访问 API 时报错：
```
Http failure response for https://119.45.170.4:44354/api/abp/application-configuration: 0 undefined
```

### 原因分析
生产服务器没有有效的 HTTPS 证书，使用自签名证书无法被浏览器接受。

### 解决方案
改为使用 HTTP 协议：

1. **修改 Angular 环境配置** (`dynamic-env.json`):
```json
{
  "oAuthConfig": {
    "issuer": "http://119.45.170.4:44354/",
    "redirectUri": "http://119.45.170.4"
  },
  "apis": {
    "default": {
      "url": "http://119.45.170.4:44354"
    }
  }
}
```

2. **更新容器内配置**:
```bash
docker exec knowledgehub-angular sh -c 'cat > /usr/share/nginx/html/dynamic-env.json << EOF
{新配置内容}
EOF'
```

## 问题 4: CORS 配置导致跨域请求失败

### 问题描述
API 请求被 CORS 策略阻止。

### 原因分析
`docker-compose.yml` 中 CORS 配置只允许 `localhost:4200`，没有包含生产服务器 IP。

### 解决方案
启动 API 容器时添加正确的 CORS 配置：
```bash
docker run -d --name knowledgehub-api \
  -e 'App__CorsOrigins=http://localhost:4200,http://119.45.170.4' \
  -e 'App__AngularUrl=http://119.45.170.4' \
  -e 'App__RedirectAllowedUrls=http://119.45.170.4' \
  ...
```

## 问题 5: OpenIddict 客户端未创建

### 问题描述
登录时报 HTTP 500 错误，数据库中 `OpenIddictApplications` 表为空。

### 原因分析
DbMigrator 运行时没有正确配置 OpenIddict 客户端的回调 URL。

### 解决方案
运行 DbMigrator 时传入正确的环境变量：
```bash
docker run --rm --name knowledgehub-db-migrator \
  -e 'OpenIddict__Applications__KnowledgeHub_App__ClientId=KnowledgeHub_App' \
  -e 'OpenIddict__Applications__KnowledgeHub_App__RootUrl=http://119.45.170.4' \
  -e 'OpenIddict__Applications__KnowledgeHub_Swagger__ClientId=KnowledgeHub_Swagger' \
  -e 'OpenIddict__Applications__KnowledgeHub_Swagger__RootUrl=http://119.45.170.4:44354' \
  -e 'ConnectionStrings__Default=Host=postgres;Port=5432;Database=KnowledgeHub;Username=postgres;Password=postgres' \
  mycompany/knowledgehub-db-migrator:latest
```

## 问题 6: API 容器缺少 uploads 目录

### 问题描述
API 启动失败：
```
System.IO.DirectoryNotFoundException: /app/uploads/
```

### 原因分析
容器内没有创建 `/app/uploads` 目录，且卷挂载路径错误。

### 解决方案
正确挂载卷：
```bash
docker run -d --name knowledgehub-api \
  -v /home/ubuntu/knowledgehub/uploads_data:/app/uploads \
  ...
```

## 注意事项

1. **HTTPS 证书**: 生产环境建议配置正规的 SSL 证书，避免使用 HTTP
2. **环境变量**: 确保所有 URL 配置一致（issuer、CORS、redirectUri 等）
3. **数据库迁移**: 每次部署后需要运行 DbMigrator 确保 OpenIddict 客户端正确配置
4. **卷挂载**: 确保 `/app/uploads` 目录正确挂载
