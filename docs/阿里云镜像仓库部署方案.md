# 基于阿里云 Docker 仓库的部署方案

## 概述

本方案支持模块化部署，可以按需更新特定服务（前端、API、数据库迁移等），无需每次全量部署。

**架构：**
```
本地开发机                    阿里云镜像仓库                  生产服务器
    │                              │                            │
    ├─ 构建镜像 ──────────────────>│                            │
    │                              │                            │
    ├─ Push 镜像 ─────────────────>│                            │
    │                              │                            │
    ├─ SSH 触发 ───────────────────────────────────────────────>│
    │                              │                            │
    │                              │<──── Pull 镜像 ────────────┤
    │                              │                            │
    │                              │        ┌─ Docker Compose ──┤
    │                              │        │                   │
    │                              │        ├─ Angular          │
    │                              │        ├─ API              │
    │                              │        ├─ Meilisearch      │
    │                              │        ├─ PostgreSQL       │
    │                              │        └─ Redis            │
```

registry.cn-zhangjiakou.aliyuncs.com/myelixir/knowledgehub-angular:latest

## 快速开始

### 1. 配置凭证

创建 `~/.knowledgehub-deploy.env`:

```bash
# 阿里云 Docker 仓库
REGISTRY_USERNAME=your_username
REGISTRY_PASSWORD=your_password

# 服务器 SSH（已在脚本中配置，可选覆盖）
# REMOTE_HOST=119.45.170.4
# REMOTE_USER=ubuntu
# REMOTE_PASSWORD=your_ssh_password
```

加载环境变量：
```bash
source ~/.knowledgehub-deploy.env
```

### 2. 部署命令

```bash
cd etc/docker

# 交互式菜单
./deploy-to-registry.sh

# 命令行模式
./deploy-to-registry.sh --all        # 部署所有模块
./deploy-to-registry.sh --angular    # 只部署前端
./deploy-to-registry.sh --api        # 只部署 API
./deploy-to-registry.sh --migrate    # 执行数据库迁移
./deploy-to-registry.sh --status     # 查看服务状态
./deploy-to-registry.sh --logs       # 查看日志
./deploy-to-registry.sh --restart    # 重启服务
```

## 详细说明

### 部署流程

#### 部署前端（Angular）

```bash
./deploy-to-registry.sh --angular
```

1. 本地构建 Angular 项目（`ng build --configuration production`）
2. 构建 Docker 镜像（`knowledgehub-angular:latest`）
3. Push 到阿里云镜像仓库
4. SSH 到服务器，拉取最新镜像
5. 重启 `knowledgehub-angular` 容器

#### 部署后端（API）

```bash
./deploy-to-registry.sh --api
```

1. 本地构建 .NET API 项目
2. 构建 Docker 镜像（`knowledgehub-api:latest`）
3. Push 到阿里云镜像仓库
4. SSH 到服务器，拉取最新镜像
5. 重启 `knowledgehub-api` 容器

#### 执行数据库迁移

```bash
./deploy-to-registry.sh --migrate
```

1. 拉取最新的 `db-migrator` 镜像
2. 执行一次性迁移任务
3. 创建/更新 OpenIddict 客户端

### 手动操作

#### 手动登录镜像仓库

```bash
docker login registry.cn-zhangjiakou.aliyuncs.com
# Username: your_username
# Password: your_password
```

#### 手动拉取镜像

```bash
docker pull registry.cn-zhangjiakou.aliyuncs.com/myelixir/knowledgehub-api:latest
docker pull registry.cn-zhangjiakou.aliyuncs.com/myelixir/knowledgehub-angular:latest
docker pull registry.cn-zhangjiakou.aliyuncs.com/myelixir/knowledgehub-db-migrator:latest
```

#### 手动启动服务

```bash
cd /home/ubuntu/knowledgehub
docker-compose -f docker-compose.registry.yml --env-file .env.production up -d
```

#### 查看日志

```bash
# 所有服务
docker-compose -f docker-compose.registry.yml logs -f

# 特定服务
docker-compose -f docker-compose.registry.yml logs -f knowledgehub-api
docker-compose -f docker-compose.registry.yml logs -f knowledgehub-angular
```

## 故障排查

### 服务无法启动

```bash
# 查看容器日志
docker logs knowledgehub-api --tail 100
docker logs knowledgehub-angular --tail 100

# 检查容器状态
docker ps -a

# 重启单个服务
docker restart knowledgehub-api
```

### 数据库连接失败

```bash
# 检查 PostgreSQL 是否正常
docker exec -it postgres psql -U postgres -c "SELECT 1"

# 检查网络
docker network inspect knowledgehub_abp-network
```

### 前端无法访问 API

1. 检查 `dynamic-env.json` 中的 API URL
2. 检查 CORS 配置
3. 检查防火墙端口（80, 44354）

## CI/CD 集成

### GitHub Actions

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to Production
        env:
          REGISTRY_USERNAME: ${{ secrets.REGISTRY_USERNAME }}
          REGISTRY_PASSWORD: ${{ secrets.REGISTRY_PASSWORD }}
        run: |
          cd etc/docker
          chmod +x deploy-to-registry.sh
          ./deploy-to-registry.sh --all
```

### GitLab CI

```yaml
deploy:
  stage: deploy
  script:
    - cd etc/docker
    - chmod +x deploy-to-registry.sh
    - ./deploy-to-registry.sh --all
  only:
    - main
  variables:
    REGISTRY_USERNAME: $CI_REGISTRY_USER
    REGISTRY_PASSWORD: $CI_REGISTRY_PASSWORD
```

## 注意事项

1. **首次部署**需要先执行数据库迁移：
   ```bash
   ./deploy-to-registry.sh --migrate
   ```

2. **更新前端**时，确保 `dynamic-env.json` 配置正确

3. **更新 API** 后，可能需要重新执行迁移（如果有数据库变更）

4. **敏感信息**不要提交到 Git，使用 `.env.production` 管理密码和密钥

5. **数据备份**：定期备份 PostgreSQL 数据
   ```bash
   docker exec postgres pg_dump -U postgres KnowledgeHub > backup.sql
   ```
